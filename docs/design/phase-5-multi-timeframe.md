# Phase 5 设计文档：多时间框架策略

## 目标

从单一 `1h` 均线交叉升级为多周期共振。

## 范围

- `15m` 短线动量
- `1h` 主决策
- `4h` 趋势过滤

## 核心设计

- 大周期负责方向
- 中周期负责入场
- 小周期负责时机优化和风险过滤

## 策略拆分

- `4h`：
  - 使用独立的快慢均线判断大趋势
  - `fast >= slow` 视为 `uptrend`
  - `fast < slow` 视为 `downtrend`
- `1h`：
  - 继续作为主决策周期
  - 只在主周期产生有效交叉时进入买卖决策
- `15m`：
  - 作为入场动量过滤
  - 买入时要求短周期快线不弱于慢线
  - 持仓时若趋势转弱且短周期动量反向，可提前触发卖出

## 入场条件

- `1h` 主周期出现金叉
- 当前无持仓
- `4h` 趋势为 `uptrend`
- `15m` 动量状态不弱于慢线
- P1/P2/P3/P4 的风控、调度、AI 闸门仍全部生效

## 卖出条件

- `1h` 主周期死叉
- 或持仓期间 `4h` 趋势转弱且 `15m` 动量转空
- 或 P2 的止损、止盈、跟踪止损、持仓超时先触发

## 产出要求

- 周期报告必须带出：
  - `signal_regime`
  - `entry_interval_summary`
  - `trend_interval_summary`
- 看板必须能显示：
  - `15m / 1h / 4h` 三层结构
  - 主信号原因
  - 当前市场结构

## 验证

- 单元测试覆盖：
  - 三周期共振买入
  - 趋势过滤拦截买入
  - 持仓中的多周期卖出
- 主流程验证：
  - `latest_report.json` 中出现 `mtf_*` 原因字符串
  - 看板能显示多时间框架结构模块
